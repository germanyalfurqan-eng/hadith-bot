# 🗂 MUSLIMOON BOT — сводка проекта (для восстановления / нового Claude)

> Этот файл + репозиторий `germanyalfurqan-eng/hadith-bot` позволяют пересобрать приложение.
> Если приложение/аккаунт потеряны — дай этот файл новому Claude вместе с содержимым репозитория.

## Что это
Telegram Mini App + бот: поиск/чтение Корана и хадисов, библиотека Мактаба аш-Шамиля (8589 книг),
наука хадиса (мусталях). Цель — база хадисов книги «الموحد المهيمن» (автор مرشد بن يوسف) с привязкой к
первоисточникам и оценкой достоверности.

## Архитектура (где что)
- **Фронтенд:** один файл `miniapp/index.html` (~5000 строк, vanilla JS, Telegram WebApp API). Деплой:
  копия в `docs/index.html` репозитория (GitHub Pages). Версия в `APPVER` + `#verBadge` + `docs/ver.txt`.
- **Бэкенд:** `bot.py` (aiohttp на Railway), эндпоинты `/api/...`, гейт по initData/HMAC + feature_allowed +
  rate_ok. ИИ — DeepSeek (`ask_deepseek`).
- **«База данных»:** ветка `data` GitHub (через `_data_get`/`_data_put`): доступы, переводы, кэши ИИ,
  обратная связь, журнал ошибок, чёрный список.
- **Данные приложения (docs/):** quran.json, quran_ru, quran_roots.json, catalog.json (8589 книг),
  books_map.json, arabus.json, reverse_index.json, hadith/* (наука хадиса).
- **Деплой-процедура:** править `miniapp/index.html` → копировать в clone `%TEMP%/hadith-bot-push/docs/` →
  поднять 3 версии → update_note.txt → commit (`-c user.name=muslimoon-bot -c user.email=noreply@anthropic.com`)
  → push → бэкап в Google Drive LATEST. Тесты бэкенда: `test_g9.py` (45/45).

## Ключевые ИИ-эндпоинты bot.py
neuro (умный поиск хадис/аят + тематические аяты), booksearch (книга/автор по-русски, режим author),
booktrans (перевод названий, кэш booknames.json), bookinfo (описание книги: автор/годы/составлена/среда/оценка),
authorinfo (биография автора), narrator_rijal (джарх/тадиль передатчика, кэш narrators.json),
structure (структурировать результаты), qaudio (прокси quran.com — пословные тайминги),
wordai, explain, tashkeel, isnad_ai, arabus, book_page (turath), errlog (журнал ошибок), devfeedback.

## Главные функции (фронтенд)
Поиск хадисов/аятов (релевантность, уникальные, по смыслу-ИИ, однокоренные, огласовки, авто-исправление,
транскрипция фраз). Читалка Корана: вся сура, выбор суры списком, аудио (everyayah, чтец по умолч. Худайфи),
фоновый мини-плеер (↪ к аяту), пословный тап (корень/места/Arabus/перевод), «✨ читать по словам»
(quran.com тайминги), басмала отдельной шапкой. Мактаба: каталог 8589, дефолт «40 первоисточников»,
⭐ избранное, разделы переведены+огласованы, умный ИИ-поиск + карточка автора + автокэш, перевод названий,
описание книги/Википедия, страница автора, читалка книг turath. Наука хадиса (☰): Правило 1 + 48 мутабарун +
труды риджаль. Разбор передатчика из поиска. Кабинет владельца: доступы, бан (мультибан+по ответу+журнал ЧС),
баланс, журналы, замечания, журнал ошибок.

## Незыблемые правила проекта
1. Отвечать владельцу по-русски, показывать прогресс %.
2. **Поиск — право человека**: хочет искать по всей Мактабе — пожалуйста; подсказки (передатчик и т.п.) —
   только МЯГКОЕ предложение, не форсить.
3. ИИ-цвет (фиолетовый) — ТОЛЬКО на ИИ-функциях.
4. Любое накопление — в отдельные файлы, append-only, удалять только с разрешения владельца.
5. Эталон текста — автор; первоисточник для починки/сверки; править — только показав владельцу.
6. Деплой коммитить через Bash heredoc (не PowerShell here-string). Co-Authored-By: Claude Opus 4.8.
